# AccessBridge AI Execution Prompt

Use this prompt when working with an AI coding agent on this repository.

---

You are building AccessBridge AI for the TechEx Intelligent Enterprise Solutions Hackathon.

AccessBridge AI is an AI-powered opportunity readiness platform that helps small businesses find, qualify for, and apply to grants, contracts, supplier diversity programs, and business funding.

## Build Rules

- Follow the PRD in `/docs/PRD.md`.
- Do not overbuild.
- Prioritize a polished, working demo over broad feature coverage.
- Keep the app easy to explain in under five minutes.
- Use mock data where real integrations would slow the team down.
- Never commit API keys, private credentials, or secrets.
- Add meaningful comments only where they clarify non-obvious logic.
- Keep components small and readable.

## Preferred Stack

- React
- Vite
- TypeScript
- Tailwind CSS
- Gemini API / Google AI Studio
- Local JSON mock data

## Core Flow

1. Landing page
2. Business profile intake
3. Opportunity dashboard
4. Opportunity breakdown
5. Generated application package
6. Enterprise intelligence dashboard

## Required Demo Data

Use this default demo business:

- Business Name: Autonomyx Solutions
- Location: Riverside, CA
- Industry: Commercial Cleaning
- Years in Business: 1
- Goal: Get contracts and funding

## MVP Quality Bar

The app should feel like a real product, not a slide deck. It needs:

- Strong copy
- Clean UI
- Visible scoring logic
- Clear AI moments
- Enterprise value layer
- Smooth demo path

## Implementation Order

1. App shell and design system
2. Landing page
3. Business profile form
4. Mock opportunity data
5. Opportunity dashboard
6. Opportunity detail view
7. Application package output
8. Enterprise intelligence dashboard
9. AI integration or simulated AI fallback
10. Demo polish

## TrapLogic Workflow

- Use milestone-based work.
- Keep a devlog for major progress.
- Each milestone should have a clear outcome.
- Avoid scope creep during hackathon build.
